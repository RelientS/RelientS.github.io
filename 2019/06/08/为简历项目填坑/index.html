  • Vue和React的区别

    • Vue可以用v-model进行双向绑定
    • 改变数据方式不同，React需要setState，而Vue通过依赖追踪。
    • React需要JSX，但是只需要JS就能控制页面。

Node.js

  • express、koa和koa2的区别

    • 总言之，最主要的区别还是异步流程控制，express是通过callback（导致回调地狱），Koa采用es6的generator，Koa2通过es7的async/await方法。
    • 此外，koa还移除了router、view等功能，通过co函数将异步变同步执行。

  • Eventloop以及process.nextTick()

  • express中间件和koa2中间件

    • express中间件即app.use(middleware)。首先app内会声明一个funcs数组用来储存需要执行的中间件（函数），调用next()函数会执行funcs里的下一个中间件。app.use则会将中间件push到funcs中。
    • koa2中间件即洋葱圈模型，先像express一样执行中间件数组。直到执行完最后一个中间件，再从后往前执行未执行完的。

Webpack（不用太深入）

  • webpack的loader和plugin区别

    • loader 用于对模块的源代码进行转换。loader 可以使你在 import 或”加载”模块时预处理文件。
    • Plugin 是用来扩展 Webpack 功能的，通过在构建流程里注入钩子实现，它给 Webpack 带来了很大的灵活性。

  • webpack的作用

    • WebPack可以看做是模块打包机：它做的事情是，分析你的项目结构，找到JavaScript模块以及其它的一些浏览器不能直接运行的拓展语言（Scss，TypeScript等），并将其转换和打包为合适的格式供浏览器使用。
