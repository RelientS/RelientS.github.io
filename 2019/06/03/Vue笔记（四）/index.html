Vue笔记（四）

发表于 2019-06-02 | 更新于 2019-06-06 | 分类于 前端技术 ， Vue

这周末搞定了手头上的一大部分事情，可以继续学习Vue了。

今晚就来学习一下Vue的实例方法。

创建实例

当new Vue()时，就会创建一个新的Vue实例。如var vm = new Vue({....}),那么vm就是一个新建的Vue实例，那么实例上就可以通过原型链找到挂载在原型上的方法。一般名为vm.$xxx()。

实例属性

  • $el 用于获取Vue实例使用的根DOM元素,即el选项所指的元素DOM
  • $data 实例观察的数据对象。Vue实例代理了对其data对象属性的访问。
  • $options 用于当前 Vue 实例的初始化选项。可以调用构造器参数里与data同级的属性和方法。
  • $refs 返回有ref特性的所有DOM元素和组件实例，该类型为Object，且只读。多用于获取节点。
  • $root 当前组件树的根实例。

实例方法

  • 扩展方法
    • $mounted() 用来挂载扩展
    • $destroy() 用来卸载扩展
  • 监听方法
    • $watch(expOrFn, callback, [options]) 观察Vue实例变化中的一个表达式或计算属性函数。监视某个属性，如果被更该，将执行回调函数。第一个参数为表达式或计算属性函数，也可以是一个属性。
  • 属性方法
    • $set(target, key, value) 设置对象的属性。
    • $delete( target, key ) 删除对象的属性。
  • 事件方法
    • $on( event, callback ) 监听当前实例上的自定义事件。事件可以由vm.$emit触发。回调函数会接收所有传入事件触发函数的额外参数。
    • $emit( event, […args] ) 触发当前实例上的事件。附加参数都会传给监听器回调。
    • $once( event, callback ) 监听一个自定义事件，但是只触发一次，在第一次触发之后移除监听器
    • $off( [event, callback] ) 移除自定义事件监听器。如果没有提供参数，则移除所有的事件监听器；如果只提供了事件，则移除该事件所有的监听器；如果同时提供了事件与回调，则只移除这个回调的监听器。
  • 生命周期方法
    • $forceUpdate() 迫使 Vue 实例重新渲染。
    • $nextTick( [callback] )将回调延迟到下次 DOM 更新循环之后执行。在修改数据之后立即使用它，然后等待 DOM 更新。在DOM更新完成后再执行回调函数，一般在修改数据之后使用该方法，以便获取更新后的DOM。

总结

要多使用实例属性和方法。
