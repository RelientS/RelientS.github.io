关于学习的一些思考

发表于 2019-06-02 | 分类于 日常 ， 思考

最近学习的过程中，发现了一些心得感悟。想要继续深挖下去，形成一套自己的学习方法论。以便未来接触各个领域的时候，有一个大致的捷径可以走。或者换句话说，就是找到一个普适的高效学习模式。学习本身是个思考的过程，但如何学习，也是一个值得思考的问题。

一步一个脚印

从小到大，我们使用的学习模式，一向是“一步一个脚印”的积累式学习。先学四则运算，再学列方程，然后学微积分。这种看似“单线程”的学习模式，加上不断地重复训练（做题与考试），确实使我们很好地掌握了基础知识。

但是这样的学习方式，在上了大学，亦或是步入社会之后，会略显疲态。因为这个时间段，我们面对的知识，已经不是线性的了，不会有人说你学完计算机基础后，必须要学C++，一定要学Java。你也可以学JavaScript，做一名前端工程师。 You can learn everything you wanna learn.

面对这样的发散式的学习选择，我们该如何用九年义务教育+高中的学习结束之后，自身具有的语数英文理五条线，来制作出一个真正强大的知识网络。这是我思考学习模式的前提。

学习如何学习

讨论学习模式的前提，要先抛开每个人的精力和学习速度差异。假定这些客观条件都相同，如何在相同的时间内，更有效率地学习就是可以讨论的了。

  1. 拒绝长时间学习同一个项目

    这其实已经是个老生常谈的问题了。人的精力是十分有限的，专注度也有一个上限。根据我的观察，很少有人能长时间高效率地去学习一件事物（至少我不行）。因此，短时间的高效学习，就显得很重要了。

    思考的一个很好的方式，就是举栗子，不能不举，举得越久越好。普适的观点，都是从一个个实例中总结出来的。用程序的思想来说，就是抽象思维。

    那么我就拿学习前端的经历来举例。一开始，我的学习方式是，先看文档，把好几十页的基础知识看完之后再动手写代码。这样带来的问题就是，比如正在学数组的内容：开始五分钟，我记住了push、pop等等这些基础方法。但是二十分钟之后，我的效率开始下滑，但这个时候我接着学习的是更深入更难的知识，所以这就导致我的学习效率非常低下，对学习欲的打击也非常大。

    针对这个问题，我开始进行分析。问题主要出在以下方面：

    • 长时间的学习同一件事物，乏味且消耗精力 = 外耗
    • 在看文档时，只是浅层记忆，没有完全理解 = 浅学习
    • 带着未完全理解的知识，继续单线程的学习深入的知识 = 内耗

    发现了问题之后，就要开始着手解决。我想出来的策略是：

    • 采用多线程，片断式学习 = 解决外耗
    • 先理解，再深入 = 盖楼
    • 学习 -> 实践 -> 理解 -> 记忆

    解释一下，这里所说的片断式学习，区别于日常的碎片式。时间长度大概在一到二小时左右，具体时间根据个人开始感到疲乏而定。当学习到开始疲乏时，停下接着摄取新知识。转头回去，花十分钟到半小时，梳理这一个小时学的知识，确保所学的内容完全理解。之后，停下手头正在学习的内容，休息半小时，再开启另一个内容。然后重复这个循环，直到大脑觉得疲惫到无法接受信息了为止。

    这样看似线性的学习，在思维网络的搭建中是不可或缺的。但线性的学习，如果多条线同时进行，这些短线在不断的学习积累过程中就会慢慢产生交集，从而交叉线组成一张知识网。

  2. 先想清楚需要学习的方向

    正如前文所说，进入社会之后，扑面而来的信息流和发展方向，让人眼花缭乱。如何选择一个适合自己的学习方向，是很一个很重要的起点。首先，先思考几个问题：

    • 我的兴趣在哪里
    • 我的目标是什么
    • 我适合做这个吗

    听得最多的一句话，就是“兴趣是最好的老师”。为什么呢？因为兴趣会给你最大的学习欲。举个栗子，很多人都喜欢打游戏，他们都对游戏很感兴趣。也就会发现，他们打起游戏来，学习效率是最高的的。因为他们专注于此，肯去思考，我要怎么完成游戏，怎么样做得比别人更好。

    所以学习的兴趣也是最重要的，选一个你很有兴趣，或是比较有兴趣但是很实用的方向，是学习效率的一个坚固的基础。

    确定了学习的大体方向之后，就要开始制定一个细节一点的学习计划。比如要学习计算机，就要学习一门语言、学习数据结构和算法、学习计算机网络等等等等。一个详细的安排，会让你的执行力和效率都大大提高。

  3. 像一棵树一样学习

    这部分是对第二点的延伸。

    自然是我们最好的指引者，也是我们类比举例的最好对象。用仿生学的方式学习，就是像树一样学习。之前我们说到，兴趣是确定学习方向的基础，那么确定了大致的学习方向之后，也就是树的主干已经长出来之后。剩余要学习的知识就像树枝一样，开始向各个方向生长。

    但是每根树枝的粗细不同，即每个需要学习的知识点权重不同。还是拿学前端来举栗子（因为我只会这个）。最重要的三根树枝，就是HTML、CSS、JavaScript。而最粗的就是JavaScript，这表示我需要花不同量的营养（时间与精力）在培育这三根树枝上。但这些都是老生常谈的问题，就不赘言了。

    主要想说一下，最近关于学习Vue的时候得到的一些感触。为什么说要像树一样学习呢，因为你得先一根枝干生长出去，自身才会有空间留给新的节点长出分支。

    详细来说，就是先触及对自己来说基本上算是崭新的领域。先学会使用模式，再去深挖其中的一些原理，有种自上而下学习的意味。这与经典的单线程“一步一脚印”的学习模式是背道而驰的。打个不恰当的比方，有点像让你先学会说Hello、Good morning这些口语，让你能和外国人交流，再来教你字母、词汇和语法这些基础知识。

    但是对我个人来说，这确实是一种很高效的学习方式。因为在涉及崭新领域的使用模式时，会让你对组成它的知识结构有一个大体的了解。比如说我刚开始用Vue-cli搭建项目的时候，其实只会JQuery，对webpack、node.js、预处理器、babel什么的一概不懂，甚至连ES6都只学到了简单的拓展阶段。但是我仅依靠着一篇《手把手教你搭建Vue-cli项目》，就做出了一个完整的作品。

    甚至做完项目之后，我也只是熟练地运用npm run dev和npm run build。但是在我的思维里，已经写好了之后的学习计划：先学ES6，然后是node.js，还有webpack等等。这些需要用到的基础知识，在生疏地使用属于“深入知识”的Vue-cli时，都在主干上有了自己的节点，甚至有一些都冒出了芽。我们接下来要做的，就是给予这些芽营养，让它们长成新的枝干就行。

    总而言之，这样的学习方式，能让你在学会基础之前，更明白该如何去学习基础，以及需要学会哪些基础。

    所以，有意识地去触碰线的末端，你才会对整条线有个大体的认识。

    知识网络的形成，最重要的不是每条线独自延伸的速度，而是形成交叉点的速度。只有把所有的知识联系起来，知识才能发挥出它的最大效用。这点可以参看学习观。
