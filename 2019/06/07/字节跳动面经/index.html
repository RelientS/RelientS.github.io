字节跳动面经（已获offer）

发表于 2019-06-07 | 更新于 2019-08-01 | 分类于 前端技术 ， 面经

9102年了，我终于也是有头条面经的人了。

一面

问了好多问题，属实榨干了。但是感觉面试官好像不太pro？问的问题都很散，导致我复盘起来十分难受。

我没做出来的题：

  1. 生成一个二叉树，实现插入和遍历功能。（我可以会，但是我没有。）

  2. 写一下[]的原型链（这我居然写了半天，属实弟弟）

  3. DNS解析域名的顺序、判断是否跨域

  4. 判断请求接口www.A.com/api 会带上哪些域的cookie

做出来的题：（榨干x2）

  1. 组合继承

  2. 写一个数组去重挂载到Array.prototype上

  3. 说一下TCP握手

  4. 跨域的三种原因（域名 协议 端口）+ 跨域的几种方式

  5. 手写AJAX

  6. 翻转链表

  7. 如何判断Array类型，instanceof的原理

  8. 0.1 + 0.2 === 0.3吗

  9. 聊项目，讲一讲websocket吧

  10. BFC 我顺便讲了IFC

二面

感觉是个厉害（装严肃的）（我讲梗居然不笑）的小哥哥来面试。

我没做出来的题

  1. jQuery里$().ready做了什么事

  2. 强缓存和协商缓存的请求头

  3. 数组扁平化（思路对了，写得很艰难）

  4. post方法都可以发送哪些数据类型

我做出来的题

  1. express中间件、实现一个服务器

  2. css题（文字省略，无限旋转动画）

  3. content-type有哪些，json和x-www-form-urlencoded的区别

  4. 有一亿条数据，需要排序找到最大的1000条

  5. express koa koa2区别

  6. 说一下gulp和webpack的区别

  7. 聊项目，各种都涉及问了一点，不深入（主要是我忘了）。

  8. 最后提问环节

我：对于现在新的技术栈不断兴起，比如可视化和全栈开发，技术点加在哪里比较好？

面试官：看个人兴趣，哪方面更感兴趣就做哪方面。比如你看如果对做游戏感兴趣，就可以去接触Unity；对可视化感兴趣，就去接触THREE等等。这些都是从图形学、图论发展出来的，也都是未来的发展方向之一，都很值得去研究和学习。不过也不要太痴迷于底层，而要想着怎么用轮子去做出更多的产品。（实用派）

面试官：好了 过十几分钟会有面试官来找你。

三面

还真就混进三面了，和做梦一样。

三面是个慈祥（？）的大叔，反正看着就像大牛23333。先跟我客套了半天，然后开始问问题。

我没做出来的题

  1. 写个eventbus（教练对不起，我用Vuex）

  2. 用promise封装一个ajax，即实现一个fetch()（这个也写得很艰难）

  3. 场景题：一个按钮按下后，一段时间内页面卡死，分析问题。

我做出来的题

  1. 说一下Vue的双向绑定机制吧（激动的搓手手，准备了这么久终于问了）

  2. 说一下webpack的原理，loader和plugin的区别，常见的有哪些

  3. 场景题：说说实现如百度搜索自动弹出补充项的场景

  4. 项目：socket.io的技术选型，如果不用这个，替代方案是什么。

  5. 说一下vue组件间的通信机制。

暂时想不起来其它了，接着继续愉快地吹水。

面试官：有什么要问我的？

我：头条工作累吗？（玩火）

面试官：项目比较多，有很多成长的机会。对我们的同学呢，也有安排培养计划，也是一项考验和磨练。你愿意接受挑战吗（手动斜眼）？

我：还是如何加技术点的那个问题。

面试官：不管加在哪里，基础首先是最重要的。其次是看个人对趋势的理解，这个没有定论。

剩下的无关紧要的吹水就不说了233。

拿到offer咯~~
